# Frontend 代码规范

## 1. 基础约定
- 必须使用 TypeScript，不允许新增纯 JavaScript 业务文件。
- 统一使用 ESLint + Prettier 自动格式化。
- 目录、文件、导出命名语义化，不使用无意义缩写。

## 2. Vue 组件规范
- 使用 `<script setup lang="ts">`。
- 单文件组件命名使用 `PascalCase`，文件名与组件名一致。
- 组件职责单一，单个组件只处理一个明确业务场景。

## 3. 状态与数据流
- 跨页面状态放入 Pinia，局部状态留在组件内。
- 禁止在组件中直接拼接后端地址，统一走 API 层。
- 后端数据必须先做类型校验与空值兜底再渲染。

## 4. 样式规范
- 样式变量集中管理，颜色与间距需可复用。
- 页面样式先保持与旧站一致，再做渐进优化。
- 禁止写行内样式覆盖核心布局，除非用于动态计算。

## 5. 地图开发规范
- 地图初始化、销毁、事件解绑必须成对出现。
- 地图点位数据通过统一转换函数映射，不在组件内散写字段。
- 地图 Key 通过配置接口注入，不写入仓库常量文件。

## 6. 可维护性要求
- 每个模块必须提供 `types.ts`、`api.ts`、`views` 或 `components`。
- 复杂逻辑提取为 composable，避免页面组件臃肿。
- 新增功能必须补充最小可用测试用例（单测或集成测试）。
