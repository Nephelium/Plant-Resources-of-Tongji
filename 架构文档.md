# Frontend 架构文档

## 1. 技术栈
- Vue 3
- Vite
- TypeScript
- Vue Router
- Pinia
- 地图 SDK 适配层（百度地图 + 兜底底图）

## 2. 架构分层
- `src/app`：应用启动、全局路由、全局状态初始化。
- `src/layouts`：与旧站一致的页面骨架（头部导航、内容区、页脚）。
- `src/modules/map`：植物地图模块。
- `src/modules/taxon`：分类系统与植物检索模块。
- `src/modules/timeline`：植物变迁模块。
- `src/modules/bbs`：留言板模块。
- `src/shared`：通用组件、API 客户端、工具函数、类型定义。

## 3. 路由规划
- `/`：植物地图（默认首页）
- `/taxon`：分类系统
- `/search`：植物检索
- `/timeline`：植物变迁
- `/bbs`：留言板
- `/bbs/:postId`：帖子详情

## 4. 页面风格保持策略
- 保留旧站背景图、主导航文案、区块布局比例与主色调。
- 统一字体优先使用微软雅黑回退链，字号保持旧站视觉节奏。
- 组件化拆分仅改变实现方式，不改变用户感知的信息层级。

## 5. 地图模块设计
- 统一 `MapProvider` 接口，屏蔽不同地图 SDK 差异。
- 功能保持：点位渲染、聚类、弹窗、搜索、定位、视野重置。
- 地图 Key 从后端配置接口获取，避免前端硬编码。

## 6. 数据交互策略
- 所有页面通过 `src/shared/api` 调用后端。
- 使用类型定义约束返回结构，避免旧站字段语义丢失。
- 关键数据支持缓存与重试，地图与检索优先保证可用性。

## 7. 组件边界
- 页面容器组件只负责流程编排。
- 业务组件负责展示与交互。
- 地图与富交互组件通过 composables 承载可复用逻辑。
